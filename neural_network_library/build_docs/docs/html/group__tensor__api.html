Detailed Description

Dense tensor primitives, shape transforms, math operations, and autograd.

Start here when working on numeric data representation and transformation. This group covers storage, indexing, element-wise and matrix operations, reductions, shape utilities, and gradient-related helpers.

See the Tensor Guide for a usage walkthrough.

Tensor Guide

Overview

nn::Tensor is the single numeric primitive used everywhere in this library. It owns a flat contiguous double buffer together with a shape vector that describes how that buffer maps to an n-dimensional array. All math operations, layer forward passes, and loss functions accept and return Tensor objects.

Creating Tensors

Three constructor patterns cover the most common needs:

#include "tensor.hpp"
using namespace nn;
// 1. Zero-initialised 2x3 matrix
Tensor a({2, 3});
// 2. From explicit values (row-major flat list)
Tensor b({1.0, 2.0, 3.0, 4.0}, {2, 2});
// 3. With gradient tracking enabled
Tensor w({0.5, -0.3}, {1, 2}, /*requires_grad=*/true);

Indexing and Shape Utilities

Flat indexing with operator[] is the fastest access path. Use at() when you have per-dimension coordinates and want bounds checking:

Tensor t({1.0, 2.0, 3.0, 4.0}, {2, 2});
double v0 = t[0]; // flat index — no bounds check, fastest
double v1 = t.at({1, 0}); // coordinate access — throws on bad index

Element-wise and Matrix Math

All binary operators require identical shapes. For matrix multiplication both tensors must be 2D:

Tensor a({2, 3});
Tensor b({2, 3});
Tensor sum = a + b; // element-wise add
Tensor diff = a - b; // element-wise subtract
Tensor prod = a * b; // element-wise multiply
Tensor quot = a / b; // element-wise divide
Tensor c({3, 4});
Tensor mm = a.matmul(c); // (2,3) x (3,4) -> (2,4)
// Scalar broadcast overloads
Tensor scaled = a * 2.0;
Tensor shifted = a + 1.0;

Gradient Tracking and Backpropagation

Set requires_grad = true on any leaf tensor whose gradient you need. Autograd records the computational graph as you call operations, then backward() walks it in reverse to accumulate .grad() on every leaf:

Tensor x({1.0, 2.0, 3.0}, {1, 3}, true); // leaf — wants grad
Tensor w({0.1, 0.2, 0.3}, {3, 1}, true); // leaf — wants grad
Tensor y = x.matmul(w); // graph edge recorded
Tensor loss = y.mean();
loss.backward(); // propagate gradients
// Inspect accumulated gradients
x.grad()->print();
w.grad()->print();

Common Pitfalls

  • Forgetting zero_grad() — gradients accumulate additively across calls to backward(). Call optimizer.zero_grad() (or module.zero_grad()) before each training step.
  • Shape mismatches — all binary ops require equal shapes; matmul requires the inner dimensions to agree. Both throw std::invalid_argument.
  • Detaching from the graph — if you want a copy without autograd overhead, construct a new Tensor from the raw data pointer or use requires_grad(false).
